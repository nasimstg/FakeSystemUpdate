# Security Policy

## Supported Versions

To ensure the security of our users, we only support the latest major version of the application.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of **FakeSystemUpdate** seriously. If you believe you have found a security vulnerability, please report it to us as described below.

### Standard Procedure

1.  **Do not open a public issue** on GitHub.
2.  Email us at **contact@softexforge.io** (or contact [Nasim STG](https://nasimstg.dev)).
3.  Include as much detail as possible:
    -   Type of issue (e.g., XSS, SQLi, Logic Flaw).
    -   Steps to reproduce.
    -   Impact of the vulnerability.
4.  We will acknowledge your report within 48 hours.

### What to Expect

-   We will investigate the issue and determine its severity.
-   We will keep you informed of our progress.
-   We will notify you when a fix is released.
-   We will credit you in our release notes (unless you prefer anonymity).

## Guidelines

-   Do not exploit the vulnerability to view or modify user data.
-   Do not perform denial-of-service attacks.
-   Do not share the vulnerability with others until it has been fixed.

Thank you for helping keep **FakeSystemUpdate** safe!
